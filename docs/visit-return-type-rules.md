# 送迎タイプ / 返却タイプ 入力ルール（那覇空港店）

**最終更新**: 2026-04-30
**対象**: 那覇空港店 全スタッフ
**対象画面**: APPデータタブ・OPシート・配車表の `送迎` / `返却` 列

---

## 📋 入力できる値は **6種類** だけ

| 表記 | 意味 | 主な利用シーン |
|---|---|---|
| **PUB** | お客様を **空港 → ヤード** へ **バス** で送迎（**Pick Up Bus**） | 那覇空港11番のりばからコースターで送迎 |
| **BDB** | お客様を **ヤード → 空港** へ **バス** で送迎（**Boarding Drop Bus**） | ヤードから空港へコースターで送る |
| **PU** | お客様を **空港 → ヤード** へ **ハイエース** で送迎 | バス満員 / 大人数で乗れない時 |
| **BD** | お客様を **ヤード → 空港** へ **ハイエース** で送迎 | バス満員 / 大人数で乗れない時 |
| **DEL** | **デリバリー（お届け）** = ホテル等にスタッフが車を届ける | お客様の指定場所にお届け |
| **COL** | **コレクション（回収）** = ホテル等にスタッフが車を回収しに行く | お客様の指定場所から回収 |
| **来店** | **送迎を希望しない** = お客様が自力でヤードまで来る | タクシー/レンタル/徒歩で来店 |
| **返却** | **送迎を希望しない** = お客様が自力でヤードに返却 | お客様自身でヤードに戻す |

---

## 🤖 GAS自動入力されるもの

予約取込時、メールの「お届け場所」「回収場所」から **GAS が自動的にセット** します：

| メール内容（場所） | 自動セットされる値 |
|---|---|
| 「那覇空港」 | **PUB / BDB** |
| 「那覇空港/レンタカー送迎バス14番のりば」 | **PUB / BDB** |
| 「送迎バス11番」「14番のりば」等 | **PUB / BDB** |
| ホテル名（例: パシフィックホテル沖縄） | **DEL / COL** |
| 住所（例: 沖縄県那覇市旭町123） | **DEL / COL** |
| 「★OTAデリバリー希望（場所未確定）」 | **DEL / COL** |

→ **基本はバス送迎前提（PUB/BDB）でセット**。ハイエース運用は手動切替。

---

## ✋ スタッフ手動入力が必要なもの

GAS が自動判定**できない**ケース → **ブランク（空欄）**で残る → **スタッフ判断で入力**

### ブランクになる主なパターン
- メールに場所情報がない予約
- 「那覇空港店」「HANDYMAN」「店舗」「来店」等が場所欄に入っている
- 「店頭」「ヤード」「営業所」などの社内表記
- お客様が場所を空欄で予約した

### スタッフが選ぶ4択

```
【ブランク予約】を見つけたら以下のどれかを判断して入力

  ① お客様が空港送迎を希望 → PUB / BDB（バス）
        ↓ バス満員 or 大人数で乗れない
  ② ハイエース運用に切替 → PU / BD
  
  ③ お客様が送迎を希望していない → 来店 / 返却
  
  ④ デリバリー希望 → DEL / COL（場所も確定して入力）
```

---

## 🚐 バス vs ハイエース の判断基準

### コースター（バス）定員: **20名 / 便**
- バス便: 9〜19時 / PU便（毎時00分発）/ BD便（毎時30分発）
- 1予約最大8名 → **単独予約で満員になることはない**

### 「PU/BD（ハイエース）」を選ぶケース
1. 同時刻帯の予約者合計が **20名超過** でバスに乗りきれない
   - APP の **🚌 バス運行表** で残席を確認
2. お客様の希望（個別送迎を希望）
3. 大型荷物等でバスに収まらない

### 「来店 / 返却」を選ぶケース
1. お客様が **送迎を希望しない**（自力来店）
2. お客様が「店頭で受け取りたい」と申告
3. お客様が「タクシーで行く」「友人が送ってくれる」等と申告

---

## 🔍 判断に迷ったときの確認ポイント

| 確認内容 | チェック方法 |
|---|---|
| 場所が空欄？ | データタブで「お届け場所」「回収場所」を確認 |
| バスが満員じゃないか？ | TOP →「🚌 バス運行表」アイコンで該当時刻の残席確認 |
| お客様の希望は？ | LINE / メール本文を確認、または電話で確認 |
| 同名の予約が他にもある？ | 検索で名前/予約番号を確認（家族予約等） |

## ⚠️ 注意事項

### GAS が触らないルール（手動値の保護）
**既に「来店」「返却」「PU」「BD」が入力済みの予約**は GAS が再パースしても **絶対に上書きしません**。安心して手動入力してください。

### バックフィル実行時のルール
過去予約を一括補正する `backfillVisitReturnType` 実行時：
- 「来店」「返却」「PU」「BD」 → **保護（触らない）**
- 「DEL」「COL」「PUB」「BDB」「ブランク」 → **place 由来で再推論**

### 入力ミス防止
- **PUB を PU と書かない**（バスとハイエースは別物）
- **「来店」を「店頭」「店」と書かない**（必ず「来店」表記）
- **DEL/COL を入力する時は場所も必ず記入**（ホテル名等）

## 📊 値の流れ（フロー図）

```
予約メール受信
    ↓
GAS自動パース（del_place/col_place から推論）
    ├─→ 空港・送迎バス系  → PUB / BDB セット
    ├─→ ホテル・住所     → DEL / COL セット
    └─→ 判別不能         → ブランク

ブランク予約をスタッフが確認
    ├─→ お客様希望が空港送迎  → PUB / BDB 入力
    │         ↓ バス満員時のみ
    │   → PU / BD 切替（手動）
    ├─→ お客様希望がデリバリー → DEL / COL 入力（場所も確定）
    └─→ お客様送迎希望なし    → 来店 / 返却 入力
```

## ❓ FAQ

**Q1. 「PUB」と「PU」の違いは？**
- **PUB** = バス（コースター）で送迎
- **PU** = ハイエースで個別送迎（バス満員時の代替）

**Q2. 同じ顧客で行きと帰りで違うタイプは可能？**
- 可能です。例: 行き「PUB」（バスで来る） / 帰り「DEL」（ホテルに届けて回収しない）

**Q3. ブランクのまま運用するとどうなる？**
- バス運行表には載りません
- OPシートには載りますが、送迎手段が不明のままになる
- → **必ず確定させてください**

**Q4. メール再取込で手動入力が消える心配は？**
- ありません。「来店/返却/PU/BD」は **GAS が絶対に上書きしない** ロジックになっています

**Q5. 「DEL」だけ入れて場所を空にしてもOK？**
- NG。デリバリーは場所がないと配車できません。**必ず場所も入力**してください
