車両タブとは

5つのサブタブで車両の全ライフサイクルを管理します

車両管理タブは、レンタカー車両の「稼働状況の把握」「コスト管理」「整備記録」をアプリひとつで完結させるための機能群です。

タブ名 主な用途 主な利用者 更新頻度
🚗 運用サマリー 今日の車両状態一覧（稼働/貸出/修理） スタッフ全員 毎日
📋 マスター 車両情報の登録・編集、ランニングコスト設定 管理者 車両追加時・変更時
🔵 車検/点検 車検・半年点検の日程確認・確定金額入力 管理者・スタッフ 車検/点検のたび
📏 距離管理 走行距離の記録、オイル交換の管理 スタッフ全員 定期的に（週1〜月1）
🔧 整備管理 整備記録の入力・費用管理 管理者・スタッフ 整備のたび
車両タブの各機能は互いに連動しています。例えば、整備管理で「車検」の金額を入力すると、車検/点検タブに自動で確定金額が反映されます。
2

運用サマリー

今日の車両状態を一目で確認する

📌 主な用途

今日現在、各車両が「稼働中（貸出可）」「貸出中」「整備中」「未稼働」のどの状態かを確認するための一覧画面です。朝のスタッフミーティングや、配車確認のときに使用します。

👁 画面の見方

表示項目 説明
稼働 N台 配車表で「稼働フラグ」が立っている車両数（★マスター設定）
未稼働 N台 配車表で稼働フラグがオフの車両（★マスター設定）
貸出中 N台 選択日時点で予約が入っている台数（配車表データ）
待機中 N台 稼働台数 − 貸出中台数（空いている車）

🔽 クラス別アコーディオン

各クラス（A/B/C/D/F/H/S）をタップすると展開します。

列 内容
車種名 / ナンバー 車両名とナンバープレート
状態 稼働 または 未稼働
次回車検 配車表に登録された次回車検日（青）
次回点検 配車表に登録された次回半年点検日（緑）
傷箇所 傷管理APPから自動取得した傷の箇所数
「日付を変更」ボタンで過去・未来の特定日付の状態を確認できます。貸出件数は選択した日付のデータを参照します。
3

マスター（車両登録管理）

車両情報の登録・編集・ランニングコスト設定

📌 主な用途

車両の基本情報（車種名・ナンバー・クラス）とランニングコスト（保険・リース・自動車税）を登録・管理します。

マスター情報は他のすべてのタブに連動します。車種名やナンバーを変更すると、配車表・整備記録・売上データにも影響します。

🚗 車両一覧の見方

クラスバッジ（A/B/C...）をタップするとそのクラスの車両一覧が展開します。デフォルトは折りたたみ状態です。

列 内容
クラス A/B/C/D/F/H/S のクラスバッジ
車種名 アルファード①、セレナH②など
ナンバー ナンバープレート番号（青字）
稼働 稼働/未稼働 の状態
保険 保険 バッジ = 保険会社所有車（自動配車除外）
月次原価 ランニングコストの月額合計（¥X,XXX）

✏️ 車両を編集する手順

1
クラスバッジ（例：[A]）をタップしてアコーディオンを展開する
2
対象車両の右側にある 「編集」 ボタンをタップ
3
モーダルウィンドウが開く。5つのタブから編集内容を選ぶ：
基本情報 車種名・ナンバー・クラスなど
保険事業 保険フラグ（保険会社所有車の場合ON）
ランニングコスト 保険料・リース代・自動車税
メンテナンス 車検・半年点検の記録確認
整備管理 整備記録の確認
4
編集後「保存」ボタンをタップ

💰 ランニングコストの設定

「ランニングコスト」タブで各車両の固定費を設定します。

項目 入力内容 月額計算方法
保険料（年額） 年間の保険料合計（例：120,000） ÷12 ヶ月
リース代（月額） 月々のリース・ローン（例：55,000） そのまま
自動車税（年額） 毎年の自動車税（例：45,000） ÷12 ヶ月
車検代（自動算出） 配車表から自動計算 ※手入力不可 ÷24 ヶ月（2年周期）
半年点検代（自動算出） 配車表から自動計算 ※手入力不可 ÷6 ヶ月（年2回）
「月額合計（概算）」は自動計算されます。この金額が経営管理タブの粗利計算に使われます。
4

車検/点検

車検・半年点検の日程確認と確定金額管理

📌 3つのサブビュー

タブ 内容
📋 車両マスター 各車両の次回車検・点検日と確定金額を一覧表示
📅 スケジュール 年別月次の車検・点検スケジュール一覧（概算・確定金額）
💰 資金繰り 当年〜来年以降の月別コスト推計

🔵 車両マスタービューの使い方

各車両の「次回車検日」「次回半年点検日」と費用（概算・確定）を確認できます。

1
クラスバッジをタップしてアコーディオンを展開
2
各車両の「次回車検」「次回点検」列で日程を確認
3
車検・点検が完了したら「確定入力」ボタンをタップして実際の費用を入力
アラートの見方：
期限超過 — 車検期限が過ぎています（要確認）
XX日後 — 30日以内に車検が来ます（準備が必要）
XX日後 — 余裕あり

5

距離管理

走行距離の記録とオイル交換管理

📌 主な用途

各車両の現在の走行距離を定期的に記録し、オイル交換のタイミングを管理します。スラックでアラートも受け取れます。

📏 走行距離を記録する

1
クラスバッジをタップしてアコーディオンを展開
2
記録したい車両の右側にある 「🚗 走行」 ボタンをタップ
3
現在の走行距離（km）・日付・メモを入力して「保存」
「最新km」列に最新の記録が表示されます。「前回km」列には1つ前の記録が常に表示され、差分（+X,XXXkm）も自動計算されます。

🔧 オイル交換を記録する

1
クラスバッジをタップしてアコーディオンを展開
2
対象車両の 「🔧 OIL」 ボタンをタップ
3
オイル交換時のkm・交換日・メモ（オイルグレード等）を入力して「保存」

🔴 オイル交換アラートの見方

バッジ 状態 対応
🔴 超過 XXXkm 目安距離をオーバー、または180日以上経過 早急にオイル交換
🟡 残 XXXkm 目安の残り25%以内、または120日以上経過 近いうちに交換予約
🟢 残 XXXkm 余裕あり 引き続き記録を続ける
⚪ 未記録 オイル交換記録なし 初回の記録を入力する

⚙️ 目安kmを変更する

各車両の「目安(km)」列にある数値の右の 「編集」 ボタンをタップすると、その車両専用の目安kmを変更できます（デフォルト：5,000km）。

📣 Slackへアラートを送信する

1
画面右上の 「📣 Slackアラート」 ボタンをタップ
2
#okinawa_operations-team に自動送信されます

送信内容には「🔴 要交換 N台」「🟡 接近 N台」「🟢 正常 N台」と平均走行距離が含まれます。

6

整備管理

整備記録の入力・費用管理・コスト集計

📌 主な用途

車検・修理・オイル交換などの整備費用を車両ごとに記録し、年間コストを把握します。ここが車両支出の一元管理場所です。

🔽 車両一覧の構成

クラスバッジをタップして展開すると、各車両の状態と整備費用が一行で確認できます。

列 内容
車両 車両名とナンバー（タップで詳細へ）
状態 稼働中修理中修理待機中
車検 / 半年点検 / 修理 / OIL / その他 当年の各カテゴリ費用合計
合計 当年の整備費合計
件数 整備記録の件数

✍️ 整備記録を入力する

1
クラスバッジをタップ → 対象車両の行をタップして詳細画面へ
2
「整備記録」セクションのフォームに記入：
種別（車検/半年点検/修理/オイル交換/その他）
日付（整備実施日）
金額（実費の円数）
詳細（必須：ブレーキパッド交換、等）
3
「整備記録を追加」ボタンをタップして保存
車検・半年点検の金額を入力した場合：自動的に「車検/点検タブ → 車両マスター」の確定金額欄に反映されます。二重入力は不要です。

📊 コスト集計の見方

各クラスヘッダーの下に「小計行」が常時表示されます：

整備管理 — コスト一覧
A Aクラス 18台 ¥419,966 ▼
A 小計 車検 ¥263,692 半年点検 ¥156,274 ¥419,966

クラスが折りたたまれていても小計は常に表示されるので、すぐに費用確認ができます。

機能の連動関係

データがどのように繋がっているかを理解する

🔗 全体のデータフロー

入力場所 データ 自動反映先
配車表（タイムライン） 車検/半年点検ブロック + 概算予算 車検/点検タブ（スケジュール・資金繰り）
マスター → ランニングコスト（月額自動算出）
整備管理 → 整備記録 車検/半年点検の実費金額 車検/点検タブ → 車両マスター（確定金額）
経営管理 → 解析 E. 車両別P&L（支出）
距離管理 → 🔧 OIL記録 オイル交換実施km・日付 マスター → メンテナンスタブ（実績自動表示）
距離管理 → アラートバッジ更新
マスター → ランニングコスト 保険料・リース代・自動車税 経営管理 → 解析 E. 車両別P&L（原価）
予約データ（OTA/HP） 売上・配車情報 経営管理 → 解析 E. 車両別P&L（売上）

💹 粗利の計算式

粗利 = 売上 − 原価（固定） − 支出（変動）
売上　　　= 予約データ × 配車表から自動集計
原価(固定) = 保険/12 + リース + 車税/12 + 車検/24 + 点検/6
支出(変動) = 整備管理の整備記録合計

粗利は「経営管理タブ → 解析タブ → E. 車両別P&L」で確認できます。

8

よくある操作手順

ケース別チェックリスト

📋 ケース①：車検を実施した後

1
整備管理 → 対象車両の詳細 → 整備記録に「車検、金額、日付」を入力
2
→ 車検/点検タブ の確定金額に自動反映されるのを確認
3
配車表 → 次回の車検ブロックを登録（概算予算も入力）

📋 ケース②：オイル交換を実施した後

1
距離管理 → 対象車両の「🔧 OIL」ボタンをタップ
2
交換時のkm・日付・メモ（オイルグレード等）を入力して保存
3
アラートバッジが🟢に変わるのを確認
4
（任意）整備管理 → 整備記録に「オイル交換、費用」も入力するとコスト集計に含まれる

📋 ケース③：修理が発生した後

1
配車表 → 対象車両のセルをタップ → メンテナンス登録 → 種別「修理」で登録（期間・概算）
2
修理完了後、整備管理 → 詳細 → 整備記録に実費を入力
3
配車表のメンテナンスブロックの終了日を実際の完了日に更新

📋 ケース④：新しい車両を追加した後

1
マスター → 右上の「＋ 車両追加」ボタン → 基本情報を入力して保存
2
編集画面の「ランニングコスト」タブ → 保険料・リース代・自動車税を入力
3
配車表 → 車検ブロックを登録（初回車検日・概算予算）
4
距離管理 → 「🚗 走行」ボタンで現在の走行距離を初回記録

📋 ケース⑤：月次の運用確認

1
距離管理 → 各車両の走行距離を更新、オイル交換アラートを確認
2
車検/点検 → 資金繰りタブ → 翌月以降の車検・点検コストを確認
3
整備管理 → コスト集計 → 当月の整備費用を確認
