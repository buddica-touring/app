🏠 1. 基本操作

ログインする

ユーザー名と4桁パスコードでAPPにログインする。

場所: 起動画面
  1. APP を開く（ホーム画面アイコン or ブラウザ）
  2. 一覧から自分のユーザー名を選ぶ
  3. 4桁のパスコードを入力
  4. TOP（🏠）画面が表示されれば成功
⚠️ パスコードは個人ごとに異なる。他人のアカウントでログインしない。
タブの構成を理解する

PCは左サイドバー、スマホは上部の横スクロールバーでタブを切り替える。

場所: 全画面共通
  1. 主要タブ: 🏠TOP / 📁CSV / 👥スタッフ / 📅出勤簿 / ⏰給与 / 📋勤怠管理 / 🚗配車 / 💳決済 / 🔧車両 / 🏆車両ランキング / 🏭入庫 / 🅿️駐車場 / 🧾会計 / 📊予算実績 / 👑顧客 / ✅タスク管理 / 🪪免許証 / 📋データ / 📜過去 / 🌍外国人
  2. TOP画面は全機能への「ランチャー（入口一覧）」になっている
  3. 外部リンク（勤怠管理・免許証など）は別タブで開く
💡 迷ったらまず 🏠TOP に戻る。カテゴリ別カードから目的の機能へ飛べる。
最新版に更新する（キャッシュ更新 / PWA）

画面が古い・表示がおかしい時にキャッシュを消して最新版を読み込む。

場所: 🏠TOP 画面上部
  1. 右下のバージョン表記（例 v1.0.xx-BT）を確認
  2. TOP上部の「🗑️ キャッシュ全消去」ボタンを押す
  3. 確認ダイアログで「OK」
  4. 自動でリロードされ最新版になる（DBデータは消えない）
💡 ホーム画面に追加（PWA）するとアプリのように起動できる。Safariの共有→「ホーム画面に追加」。

📥 2. 予約管理

OTA予約をCSVで取り込む

OTA（じゃらん/楽天など）からダウンロードした予約CSVを一括取込する。

場所: 📁CSV タブ
  1. 📁CSV タブを開く
  2. CSVファイルを選択（OTAごとに対応）
  3. 差分プレビューで新規・変更を確認
  4. 取込実行 → 件数・売上・OTA別内訳が表示される
  5. 保存完了まで約15秒待つ（DB保存中）
⚠️ 取込後は右上のバージョン表記が出ていることを確認。乗車人数は最大8人で自動クランプされる。
予約を確認・直接編集する

予約マスター（全予約）を一覧で確認し、必要に応じて直接編集する。

場所: 📋データ タブ
  1. 📋データ タブを開く
  2. 日付フィルタで対象期間を絞る
  3. 予約行をクリックして内容を確認
  4. 料金・場所・担当などを直接編集して保存
⚠️ 予約データ（reservations）が唯一のマスター。配車表・OPシート・データタブは常に同じ情報を表示する。
予約をキャンセルする

予約を取消（論理キャンセル）する。誤操作防止の2段階確認あり。

場所: 📋データ / 🚗配車 / OPシート
  1. 対象予約を開く
  2. CXL（キャンセル）操作を選ぶ
  3. 確認ダイアログ → さらに予約番号入力で完全一致確認
  4. キャンセル確定（配車も解除される）
⚠️ キャンセルは元に戻しにくい。スマホ誤タップ防止のため2段階確認が必須になっている。
キャンセル一覧を見る・予約を復元する

キャンセル済みの予約だけを一覧表示し、必要なら confirmed に復元する。

場所: 🏠TOP → 「❌ キャンセル一覧」タイル
  1. TOPの「❌ キャンセル一覧」タイルをタップして開く
  2. 期間フィルタ（全期間／今月／今年・返却日基準）で絞り込む
  3. 予約者名 / 予約番号で検索する
  4. 上部にキャンセル件数と売上合計（赤系）が出る
  5. 各行の「↩︎ 復元」で confirmed に戻す（確認ダイアログあり・戻すと一覧から消える）
⚠️ 復元は status を confirmed に書き換える操作。誤復元しないよう、確認ダイアログで対象を確かめてから実行する。キャンセルが無い期間は「キャンセルはありません」と表示される。

🚗 3. 配車・当日業務

配車タイムラインで車両を割り当てる

予約に対して空車をドラッグで配車する（月/週/日表示）。

場所: 🚗配車 タブ
  1. 🚗配車 タブを開く
  2. 月/週/日 で表示を切替
  3. 未配車の予約をドラッグして空車ラインへドロップ
  4. 青帯（整備）・赤帯（別予約）のラインには配車しない
⚠️ 空車がない場合「配車できる空車がありません」と出る。重複配車は禁止。
OPシートで当日業務を確認する

当日の出発/返却業務をフルスクリーンで管理する（DEL/COL/洗車タブ）。

場所: 🏠TOP →「📋 OPシートを開く」
  1. TOPの「📋 OPシートを開く」ボタンを押す
  2. DEL（配達）/ COL（回収）/ 洗車 タブを切替
  3. 担当・PU/BD・場所などを入力
  4. ドラッグで並び替え可能
⚠️ PU=空港出発（緑）、BD=ヤード出発（赤）。方向を逆にしない。担当などの入力は消さない。
タスク管理を使う

DEL/COL/事前駐車/乗捨回収/送迎/入庫/引取/その他のタスクを管理する。

場所: ✅タスク管理 タブ
  1. ✅タスク管理 タブを開く
  2. 当日のタスク種別を確認
  3. 担当を割り当て・完了チェック
入庫・整備カレンダーを管理する

車検・点検・整備の入庫スケジュールと請求書を管理する。

場所: 🏭入庫 タブ（intake.html）
  1. 🏭入庫 タブを開く
  2. 📅入庫スケジュール / 📜入庫リスト / 🚨車検・点検 / 🧾請求書 で切替
  3. カレンダーは週表示・デフォルト展開
  4. 車検・点検アラートで期限が近い車両を確認

💳 4. 決済・会計

じゃらん事前決済を確認する

じゃらん予約の事前決済状況を確認・消し込みする。

場所: 💳決済 タブ
  1. 💳決済 タブを開く
  2. 未完了の決済件数を確認（TOPにもバッジ表示）
  3. 入金済みを確認して消し込み
会計（現金出納・立替・予約外売上）を入力する

現金出納帳・立替金・予約外売上を記録する。

場所: 🧾会計 タブ
  1. 🧾会計 タブを開く
  2. 収入/支出/立替/予約外売上 を入力
  3. 立替金は回収状況を管理（未回収アラートあり）
  4. 月初残高を設定して残高を管理
💡 予約外売上（extra_sales）は経営ダッシュボードの「予約外」に自動反映される。

🔧 5. 車両

車両マスターを管理する

車両の登録・編集・クラス設定・固定費（リース/保険/車検/点検）を管理する。

場所: 🔧車両 タブ
  1. 🔧車両 タブを開く
  2. 車両を追加・編集（クラス AA/A/B/C/S/H/F）
  3. リース月額・保険・車検費・点検費などを入力
  4. 保険車両フラグ・稼働フラグを設定
  5. 整備履歴：受領請求書から連携した記録は 📥連携 バッジ＋📄 請求書 ボタンで連携元の請求書(画像/PDF)を直接開けます（受領請求書タブで画像登録済みの場合）。
⚠️ 車両マスターの変更は配車表・OPシート・ランキングなど全領域に波及する。
車両ランキングを見る

全車両の売上・稼働率・粗利を点数化してS/A/B/C/Dでランク表示する。

場所: 🏆車両ランキング タブ
  1. 🏆車両ランキング タブを開く
  2. クラス別ランキングで集計を確認
  3. 全車両ランキングで①合計売上②月平均③稼働率④固定費⑤予想粗利を確認

👥 6. スタッフ

スタッフ情報・時給を設定する

スタッフの登録・時給設定を行う。

場所: 👥スタッフ タブ
  1. 👥スタッフ タブを開く
  2. スタッフを追加・編集
  3. 時給などを設定
出勤簿（シフト）を作成する

シフト表を作成・編集する。TOPに本日出勤者が表示される。

場所: 📅出勤簿 タブ
  1. 📅出勤簿 タブを開く
  2. 日付ごとにシフトを入力（出勤/公休/希望休/有給など）
  3. 勤務時間を入力
給与を確認する

勤怠を元に給与（予算・実績・明細）を計算する。

場所: ⏰給与 タブ
  1. ⏰給与 タブを開く
  2. 対象月を選ぶ
  3. 実績・明細を確認
勤怠管理（有給・希望休・残業）を見る

有給・希望休・残業集計を専用ページで管理する。

場所: 📋勤怠管理 タブ（kintai.html・別タブ）
  1. 📋勤怠管理 タブを開く（別タブで開く）
  2. 有給・希望休・残業を集計確認
シフト試算（バイト予算・人件費）を使う

面接で聞いた個人条件（時給・勤務可能曜日・曜日別の勤務時間・希望週日数）を入力し、カレンダーにシフトを配置して、月間人件費・年間コスト・人件費率・シフト充足率を試算する。採用判断・予算組みに使う。

場所: 🧮シフト試算 タイル（スタッフ管理・shift-sim.html・別タブ）
  1. 🧮シフト試算 タイルを開く（別タブ）
  2. 「店舗・予算の前提」を自店に合わせて入力（営業時間/必要人数/会社負担%/最低賃金/予算/売上見込み）
  3. 「スタッフを追加」→ 氏名・時給・交通費・曜日ごとの勤務時間を入力（空欄＝休み。曜日名タップでON/OFF切替・平日のみ/週末のみ等のプリセットあり）
  4. 曜日ごとの勤務時間を設定すると自動でカレンダーに反映（ボタン操作不要）。個別に増減したい日はカードを選択して日付クリックで手動調整（「🔄 条件でこの月を再生成」で条件どおりに戻せる）
  5. 下部の「この月の試算」で月間人件費・予算差・人件費率・スタッフ別コストを確認

※入力はブラウザに自動保存（localStorage）。前提・初期値は仮値なので実態に合わせて変更する。最低賃金を下回る時給は赤字で警告。

📊 7. 経営・分析・プライシング

経営ダッシュボードを見る

売上4分解・稼働率・年間推移など経営KPIをまとめて確認する（オーナー専用）。

場所: 経営タブ → 📊ダッシュボード
  1. TOP「データ・分析」→「🔒経営」を開く
  2. サブタブ: 🎯KPI管理表 / ⚙️設定 / 📊ダッシュボード / 📝売上入力 / 📘月次PL/CF / 🔵HP-CV / 📈投資回転率 / 🔬解析 / 🩺データ健全性
  3. 📊ダッシュボードで売上4分解（基本/付帯/予約外/割引/合計）・稼働率を確認
  4. 月/年トグルで期間切替
💡 KPIは計算し直さず、この画面の表示値をそのまま使う（数字をブレさせない）。
解析（稼働率・売上・OTA・P&L）を見る

稼働率・売上構成・OTA別・車両別P&Lなどを詳細分析する。

場所: 経営タブ → 🔬解析
  1. 経営タブ → 🔬解析 を開く
  2. A.稼働率 / B.予約・売上 / OTA別構成 / 車両別P&L などのセクションを開く
  3. 期間フィルタで対象月を選ぶ
予算実績（PL/CF・コスト）を見る

予算と実績を比較し、PL/CF・コスト内訳を確認する。

場所: 📊予算実績 タブ（monthly.html / costmatrix.html）
  1. 📊予算実績 タブを開く
  2. 予算 vs 実績の予実比較を確認
  3. コスト内訳（costmatrix）を確認
売上予測シミュレータを使う

5KPI×12ヶ月のアンサンブル予測で売上着地を見込む（株価チャート風）。

場所: TOP「📈 実績/予測チャート」/ forecast.html
  1. TOPの「📈 実績/予測チャート」を押す（別タブ）
  2. 過去実績・当月予約率・先行予約から着地を予測
  3. 当月発生売上・当月予約率も確認できる
💡 当月予約率＝当月返却のうち直前予約の割合。月途中の着地予測に使う。
収支シミュレータを使う

クラス×台数×単価×稼働で収支・達成見込みをシミュレーションする。

場所: sim.html（収支シミュレータ）
  1. 収支シミュレータを開く
  2. クラスごとの単価・稼働・OTA構成比を入力
  3. OTA手数料が自動計算され、達成見込みA〜Cが出る
価格表（シーズナルカレンダー）を見る

クラス別・日別のシーズナル価格をカレンダーで確認する（表示専用）。

場所: 🗓️価格表 タブ（pricing.html）
  1. 🗓️価格表 タブを開く
  2. 年/月タブでカレンダーを切替
  3. シーズナル区分（A繁忙/B通常/C閑散）とクラス別価格を確認
⚠️ 価格表は表示専用。編集は「価格戦略マスター」で行う。
価格戦略マスターで価格を設定する

価格の編集・5バージョン履歴管理を行うマスター画面（プライシングシステム）。

場所: import-prices.html（価格戦略マスター）
  1. 価格戦略マスター（💹マスター）を開く
  2. 月別・クラス別に価格を編集
  3. セルの 💾 ボタン または Enter で明示保存
  4. V1=原案（不変）/ V2〜V5=保存スナップショット。採用中バージョンは緑枠
  5. 📅シーズナル設定（seasonal-config.html）で繁忙/閑散期間を設定
⚠️ 保存は 💾 / Enter のみ（blurでは保存しない）。価格表側は表示専用なので編集はここで行う。
その他の分析タブ

顧客・外国人利用率・免許証・過去タスクなどの分析・管理。

場所: 各タブ
  1. 👑顧客: リピーター分析
  2. 🌍外国人: 外国人利用率の推移
  3. 🪪免許証: 免許証画像アップロード（license.html・別タブ）
  4. 📜過去: 過去のタスク履歴

🆘 9. トラブル対応

画面が真っ白 / 表示がおかしい

キャッシュ不整合や古いバージョンが原因のことが多い。

  1. TOPの「🗑️ キャッシュ全消去」を押す
  2. それでも直らなければブラウザを完全終了して再起動
  3. PWA（ホーム画面アプリ）なら一度削除して再追加
💡 DBデータは消えない。表示だけをリセットする操作。
予約や配車が他端末と食い違う

Realtime同期のずれ。少し待つか再読込で解消する。

  1. 右上が「🟢 同期中」になっているか確認
  2. 「🔴 未接続」なら通信環境を確認して再読込
  3. 編集直後は2秒ほど待ってから確認（同期反映待ち）
配車できない / 空車がない

空車不足・整備や別予約で埋まっている可能性。

  1. 「配車できる空車がありません」表示を確認
  2. 配車表で青帯（整備）・赤帯（別予約）の状況を確認
  3. 同クラスの空車を探す or クラス変換を検討
重複データが表示される

表示上の重複か、データの重複かを切り分ける。

  1. TOPの「🔍 重複検出」ボタンを押す
  2. data state に重複なし → 表示の問題（キャッシュ全消去で解消）
  3. 重複ID一覧が出る → データ側の問題（管理者に連絡）
